## 📌 About AVYRA  

AVYRA is an Income Stability Engine that predicts disruptions, guides workers to earn before impact, and dynamically insures only the remaining income loss.

---

## ⚙️ The 4-Layer AVYRA Engine  

Every other team detects a trigger and pays money.  
AVYRA does something fundamentally different — it operates across four intelligent layers that together ensure zero financial shock for the worker.

**Predict → Guide → Adjust → Protect**

| Layer | Name    | Engine                      | Description |
|------|--------|-----------------------------|------------|
| 01   | PREDICT | Disruption Forecast Engine | Weather + AQI + civic APIs detect disruptions in real time. Rain at 3 PM flagged by 9 AM. |
| 02   | GUIDE   | Adaptive Work Strategy     | "Rain hits at 3 PM. Work 10 AM–2 PM. Earnable today: ₹450." Worker earns before disruption. |
| 03   | ADJUST  | Dynamic Coverage Engine    | ₹700 − ₹450 earned = ₹250 remaining loss → AVYRA covers exactly ₹250. Precise and fair. |
| 04   | PROTECT | Real-Time Payout           | Coverage adapts continuously. Auto payout within 2 hours. |

## 💡 Key Insight  

Traditional insurance: disruption happens → claim filed → money paid.  

AVYRA: disruption predicted → worker guided to earn first → only the remaining loss is covered.  

💥 Same protection. Smarter system.

## ❗ Problem Statement  

India's platform-based delivery partners (Zomato, Swiggy, Zepto, Amazon, etc.) form the backbone of our digital economy.  

With over **50 million gig workers in India**, external disruptions like extreme weather, floods, pollution, and civic curfews can halt their ability to work — causing them to lose **20–30% of their monthly earnings** with absolutely **no safety net**.

## ⚡ Parametric Triggers  

---

### 💡 What is Parametric?  

A parametric trigger is an objective, data-driven rule:  
when a measured value crosses a predefined threshold, the payout fires automatically.  

❌ No human approval  
❌ No paperwork  
❌ No waiting  

👉 Like a thermostat:  
- Temperature drops → heater activates  
- Rain exceeds 40mm/hr → AVYRA payout activates  

---

### 📊 Trigger Table  

| #  | Trigger Event        | API Source              | Threshold              | Adaptive Payout |
|----|---------------------|------------------------|------------------------|-----------------|
| T1 | Heavy Rain          | OpenWeatherMap         | Rainfall > 40mm/hr     | Remaining loss |
| T2 | Extreme Heat        | OpenWeatherMap         | Temperature > 42°C     | Remaining loss |
| T3 | Hazardous AQI       | AQICN.org              | AQI > 400              | Remaining loss |
| T4 | Civic Disruption    | NewsAPI + Admin Panel  | Curfew / Section 144   | ₹700 full coverage |
| T5 | Platform Outage     | Mock API (simulated)   | Downtime > 2 hours     | Remaining loss |

---

### 🔄 Adaptive Logic  

- For **weather triggers (T1–T3, T5)**:  
  AVYRA guides the worker to earn first → then covers only remaining income loss  

- For **civic triggers (T4 — curfew)**:  
  No work possible → **100% income automatically covered**  

💥 This is what makes AVYRA truly adaptive.

## 🖥️ Platform Choice  

---

### 📱 Decision: Responsive Web Application (Mobile-First)

| Criteria          | Web App ✅ (Our Choice)        | Native Mobile App |
|------------------|-------------------------------|------------------|
| Build Speed      | Faster — single codebase       | Slower — separate builds |
| Demo / Judging   | Easy screen share              | Hard to present |
| Admin Dashboard  | Full desktop support           | Limited screen space |
| Worker UX        | Mobile-responsive (375px)      | Native feel but complex |
| Team Complexity  | One stack                     | Multiple platforms |

---

### 💡 Rationale  

We chose a responsive **React web application** for both desktop and mobile use.

- Worker experience → mobile-first (375px screens)  
- Admin dashboard → desktop-focused  

👉 This allows us to focus on the **Adaptive Parametric Engine** instead of platform complexity during the 6-week hackathon.

### 📊 What is Covered vs Not Covered
